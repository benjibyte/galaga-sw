Great — here’s a concise, step-by-step plan to build a Galaga-style game with LÖVE, compile it to WebAssembly (via Love.js), host on GitHub Pages, and add a PWA manifest + service worker for offline access.

## 1) Project structure (starter)
- Project root/
  - main.lua
  - conf.lua
  - src/ (optional modules: player.lua, enemy.lua, bullet.lua, spawn.lua, utils.lua)
  - assets/ (images, spritesheets, audio)
  - index.html (generated by love.js)
  - love.js/ (love.js runtime and build output)
  - manifest.json
  - service-worker.js

## 2) Develop in LÖVE (desktop) first
- Ensure main.lua exists and runs with `love .`.
- Implement minimal game loop:
  - love.load(): load sprites, sounds, set window size
  - love.update(dt): movement, spawn, collisions, scoring, simple AI patterns
  - love.draw(): render background, player, enemies, bullets, UI
  - love.keypressed(key): fire, start, pause
- Recommended incremental features:
  1. Player ship movement + shooting
  2. Single enemy type that moves and can be shot
  3. Enemy waves/formation spawning
  4. Simple enemy projectiles and collisions
  5. Score, lives, simple UI
  6. Powerups, multiple enemy patterns, boss (optional)
- Useful libs (pure Lua):
  - hump.timer/hump.camera/hump.vector (optional utilities)
  - bump.lua or HardonCollider for collisions (if you want AABB/shape detection)
  - anim8 for spritesheet animation

## 3) Make it web-ready with Love.js
- Love.js compiles the LÖVE runtime to WebAssembly and runs your project in the browser.
- Steps:
  1. Install love.js: grab latest release from its repo (it’s a set of JS + WASM files).
  2. Create a .love zip archive of your project root (contents of project root zipped; must contain main.lua at top level). Example:
     - From project folder:
       ```
       zip -r mygame.love . -x love.js\*
       ```
  3. Place mygame.love and love.js runtime files together in a web folder. Alternatively, use the love.js template index.html and point it to mygame.love.
  4. Test locally by serving the folder over HTTP (browsers block WASM over file://):
     ```
     python3 -m http.server 8000
     ```
     Open http://localhost:8000 and confirm the game runs.

Notes:
- Web builds must use web-safe file I/O — avoid absolute paths; use relative asset paths.
- Some Lua libraries that rely on native C will not work in love.js.

## 4) Add PWA support (manifest + service worker)
- manifest.json (example fields):
  - name, short_name, start_url, display (standalone), background_color, theme_color, icons array (PNG sizes: 192x192, 512x512).
- service-worker.js:
  - Use a simple cache-on-install approach to cache index.html, love.js, love.wasm, mygame.love, assets and manifest/icons.
  - Example flow:
    - install event: open cache, addAll([...]).
    - fetch event: respondWith(cache.match(request) || fetch(request)).
  - Ensure proper versioning of cache name and update strategy (cache-first for game assets).
- Register service worker from index.html:
  ```
  if ('serviceWorker' in navigator) {
    navigator.serviceWorker.register('/service-worker.js');
  }
  ```
- Configure start_url in manifest to include `?source=pwa` if needed.

## 5) Host on GitHub Pages
- Put the built web folder (index.html, love.js, love.wasm, mygame.love, manifest.json, service-worker.js, icons, assets) into the repo branch used for Pages (commonly gh-pages or main/docs).
- Enable GitHub Pages in repo settings.
- Test the public URL (HTTPS required for service worker/PWA). Confirm manifest and service worker are served (check DevTools Application tab).

## 6) Mobile/desktop PWA considerations
- Provide responsive canvas scaling; use CSS to make canvas fill available space or letterbox.
- Input: support touch controls (virtual buttons/joystick) and keyboard.
- Audio: browsers often block autoplay; call audio:play() in a user gesture (e.g., on first tap).
- Performance: reduce draw calls, use spritesheets, compress images and audio.

## 7) Build & automation tips
- Automate .love packaging with a simple script (zip).
- Add a GitHub Actions workflow to:
  - Zip the .love
  - Copy love.js runtime into build
  - Push the built web folder to gh-pages branch (use actions-gh-pages)
- Example Action steps: checkout, run build script, deploy to gh-pages.

## 8) Quick troubleshooting checklist (web)
- Serve over HTTP(S) — WASM won’t load from file://.
- Confirm Content-Type headers if hosting manually (application/wasm).
- Open browser console for love.js errors (missing mygame.love, wasm runtime load error, asset 404s).
- Ensure main.lua is top-level in the .love zip.
